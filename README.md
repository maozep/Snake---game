# Verilog Snake – Tang Nano 9K

Classic Snake game implemented entirely in Verilog, running on a **Gowin GW1NR-9C (Tang Nano 9K)** FPGA with a **Sipeed 7-inch RGB LCD (800x480 @ 60 Hz)** and a **4x4 matrix keypad**.

---

## Hardware

| Component | Details |
|---|---|
| FPGA | Tang Nano 9K (Gowin GW1NR-9C, 8640 LUTs, 64 Kbits BRAM) |
| Display | Sipeed 7-inch RGB LCD 800x480 @ 60 Hz (RGB565) |
| Input | 4x4 matrix keypad (Dupont wires) |
| System clock | 27 MHz (onboard crystal, pin 52) |
| Pixel clock | 33 MHz (generated by Gowin rPLL) |

## Game Controls

```
Keypad layout:
       COL0  COL1  COL2  COL3
ROW0:   1     2     3     A       2 = UP
ROW1:   4     5     6     B       4 = LEFT    5 = PAUSE    6 = RIGHT
ROW2:   7     8     9     C       8 = DOWN
ROW3:   *     0     #     D
```

| Key | Action |
|---|---|
| **2** | Move up |
| **8** | Move down |
| **4** | Move left |
| **6** | Move right |
| **5** | Pause / Resume |
| **S1 (reset)** | Restart game |

180 U-turns are blocked (you cannot reverse into yourself in a single move).

---

## Game States

| State | Description |
|---|---|
| **ST_IDLE** | Title screen: "SNAKE" + "PRESS S1". Press any arrow key to start. |
| **ST_PLAYING** | Active game. Score displayed top-left. Speed increases with score. |
| **ST_GAME_OVER** | Red flashing screen + "GAME OVER" text. Press S1 to restart. |

## Implementation Details

### Snake Body – Circular Buffer

```
body_col[0..63]  -  column of each segment (5-bit, 0-24)
body_row[0..63]  -  row    of each segment (4-bit, 0-14)
head_ptr         -  index of the head in the buffer
snake_len        -  current number of segments
tail_ptr         =  head_ptr - snake_len + 1  (mod 64)
```

On each move:
- Increment `head_ptr`, write new head position.
- If **not eating food**: clear the tail cell.
- If **eating food**: keep the tail (`snake_len++`, `score++`).

### Presence Map – O(1) Collision & Rendering

```
body_map[row x 25 + col]  =  1  <->  cell occupied by snake
```

- Self-collision: single bit lookup, no loop.
- Rendering: `body_map[cur_cidx]` per pixel.
- Edge case: if next cell == tail cell and not eating, no collision (tail vacates same step).

### Food – 16-bit Galois LFSR

Advances every pixel clock (~33M times/sec). Food placed when `food_active=0` and candidate cell is free.

### Dynamic Speed

| Score | Frames/move | Moves/sec |
|---|---|---|
| 0-4 | 15 | 4.0 |
| 5-9 | 12 | 5.0 |
| 10-14 | 9 | 6.7 |
| 15+ | 6 | 10.0 |

### Character Font

5x7 pixel bitmaps (35-bit constants). Full set: digits 0-9 and letters A-Z.
Rendered at 2x scale (10x14 pixels per character) in the combinational rendering path.

## Build & Simulate

```bash
iverilog -g2005 -o snake_sim.vvp \
    clock_divider.v vga_controller.v keypad_scanner.v \
    snake_top.v snake_top_tb.v

vvp snake_sim.vvp

gtkwave snake_top_tb.vcd
```

Expected output:
```
--- Phase 1: ST_IDLE ---
PASS: VSYNC timing OK
PASS: Title text visible
PASS: No game pixels in IDLE
--- Phase 2: ST_PLAYING ---
PASS: Head pixel count OK
PASS: Food rendered
PASS: Score overlay rendered
PASS: No pixels outside DE
```

### FPGA Synthesis (Gowin IDE)

1. Open Gowin IDE, create project targeting `GW1NR-9C C6/I5`.
2. Add all `.v` files and `snake_top.cst`.
3. Set top module: `snake_top`.
4. Run **Synthesize -> Place & Route -> Generate Bitstream**.
5. Flash via USB with GOWIN Programmer.
